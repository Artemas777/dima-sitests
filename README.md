# Дмитрий — лендинг консультанта по банковским продуктам

Премиальный статический лендинг: чистый HTML/CSS/JS, без сборщиков и npm-зависимостей
(единственная внешняя зависимость — шрифты Google Fonts, подключённые через `<link>`).

## Структура проекта (полная, ничего не пропущено)

```
.
├── index.html          # единственная HTML-страница
├── README.md
├── css/
│   └── style.css       # ВСЕ стили в одном файле, без @import
└── js/
    └── main.js          # навигация, scroll-reveal, 3D-карта, FAQ-аккордеон
```

Всего 4 файла в 2 папках — минимальная, самодостаточная структура без служебных
скрытых файлов (никаких `.nojekyll`, `.gitkeep`, `.DS_Store` и т.п.), чтобы ничего
не «терялось» при скачивании или загрузке в репозиторий.

Важное изменение по сравнению с прошлой версией: раньше CSS был разбит на 11 файлов,
склеенных через `@import` в `css/main.css`. Это рабочий, но более хрупкий вариант —
если хотя бы один из подключаемых файлов не долетает при загрузке в репозиторий
(частая история при ручной загрузке через веб-интерфейс GitHub), вся цепочка стилей
молча ломается, и браузер показывает страницу дефолтными стилями user-agent —
именно синий подчёркнутый текст и «съехавшее» меню, которые вы видели.

Теперь всё объединено в **один файл `css/style.css`** — меньше файлов, меньше точек
отказа. `index.html` ссылается на него одной строкой:

```html
<link rel="stylesheet" href="css/style.css" />
```

## Как запустить локально

Просто откройте `index.html` в браузере — сборка не требуется.

Либо поднимите локальный сервер:

```bash
python3 -m http.server 8080
# затем откройте http://localhost:8080
```

## Как выложить на GitHub Pages — по шагам

1. Загрузите **содержимое** этой папки (а не саму папку `dmitry-landing`) в корень
   репозитория — то есть `index.html`, `css/`, `js/`, `README.md` должны лежать
   прямо в корне репо, а не быть вложены в ещё одну подпапку.
2. Settings → Pages → Source: выберите ветку (обычно `main`) и папку `/ (root)`.
3. Подождите 1–2 минуты, пока GitHub соберёт и опубликует сайт (статус сборки —
   во вкладке **Actions** репозитория).
4. Откройте сайт в приватном/инкогнито-окне или сделайте жёсткое обновление
   (`Ctrl+Shift+R` / `Cmd+Shift+R`), чтобы исключить старую версию из кэша браузера.

## Если стили всё равно не подключились — быстрая диагностика

Симптом «синий подчёркнутый текст, меню без стилей» = **браузер не смог загрузить
CSS-файл**, а не ошибка в самом CSS. Проверьте по порядку:

1. **DevTools → вкладка Network**, перезагрузите страницу, найдите `style.css`.
   Если статус `404` — путь неверный или файла нет в репозитории по этому пути.
   Кликните на запрос и посмотрите точный URL, который запрашивал браузер.
2. **Регистр букв.** GitHub Pages — это Linux-сервер, а Linux различает регистр
   в путях (в отличие от Windows/macOS). `CSS/Style.css` ≠ `css/style.css`.
   Проверьте, что папка называется именно `css` (не `Css`, не `CSS`), а файл —
   именно `style.css`.
3. **Откройте репозиторий на GitHub.com** и убедитесь глазами, что там реально
   есть папка `css` с файлом `style.css` внутри, а не так, что все файлы лежат
   плоско в корне (это случается, если при загрузке через веб-интерфейс GitHub
   перетаскивали файлы по одному, а не всю структуру папок целиком).
4. **Вложенность.** Если в репозитории `index.html` лежит внутри ещё одной папки
   (например, `dmitry-landing/index.html`), а не в корне — сайт будет открываться
   по адресу `username.github.io/repo/dmitry-landing/`, а не `username.github.io/repo/`.
   Либо переместите содержимое в корень репозитория, либо открывайте правильный URL.
5. **Jekyll-обработка (редкий случай).** GitHub Pages по умолчанию пропускает сайт
   через Jekyll. Наш проект её не требует и не должен от неё пострадать (ни одна
   папка не начинается с `_`), но если вдруг захотите полностью её отключить —
   создайте в репозитории через веб-интерфейс GitHub («Add file → Create new file»)
   файл с именем `.nojekyll` и пустым содержимым. Через веб-интерфейс GitHub такие
   файлы создаются без проблем, даже если локально их не видно как «скрытые».

## Перед публикацией — замените плейсхолдеры

- `index.html` → ссылки `href="#"` рядом с комментарием `<!-- TODO -->` у кнопок
  «Написать в Telegram» / «Написать в WhatsApp» и в футере — замените на реальные
  `https://t.me/...` и `https://wa.me/...`.
- Цифры в hero (`4+ года`, `12 банков-партнёров`, `600+ заявок`) и в блоке
  «Обо мне» — иллюстративные, замените на реальные показатели.
- Суммы вознаграждения в разделе «Вознаграждение» — проверьте актуальные условия
  перед публикацией.

## Технические заметки

- Без фреймворков и сборщиков: ванильные HTML/CSS/JS, один CSS-файл, один JS-файл.
- Анимации уважают `prefers-reduced-motion`.
- Интерактивная карта в hero не требует изображений — реализована на CSS/SVG.
- Фавикон — inline SVG (data URI) прямо в `<head>`, отдельный файл не нужен.
- Все декоративные SVG помечены `aria-hidden`, интерактивные элементы имеют
  видимый фокус (`:focus-visible`) для доступности с клавиатуры.
- Все пути в проекте относительные (`css/style.css`, `js/main.js`, `#anchor`) —
  сайт одинаково работает и как пользовательский сайт (`username.github.io`),
  и как сайт проекта (`username.github.io/repo-name`).
